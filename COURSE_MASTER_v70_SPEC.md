# COURSE MASTER v7.0 仕様書

## 概要

**COURSE MASTER v7.0** は、JRA競馬の着順予測エンジンです。12軸のスコアリング手法と4チーム合議制により、レースごとに◎1頭○3頭▲5頭の計9頭を予測します。

---

## 1. システムアーキテクチャ

### データフロー

```
結果データ（2015-2026）        血統データ（51,111馬）
      ↓                              ↓
  [データ結合]  ←──────────────────┘
      ↓
  [統計マスター構築]
  ├─ 種牡馬別統計 (652件)
  ├─ 母の父別統計 (1,117件)
  ├─ 騎手別統計 (251件)
  ├─ 調教師別統計 (262件)
  ├─ クラス別統計 (11件)
  ├─ トラック別統計 (20件)
  └─ 距離別統計 (48件)
      ↓
  [1レース予測]
  ├─ レース出馬表を入力
  ├─ 各馬について4チーム×12軸計算
  ├─ 4チームのスコアを平均化
  ├─ 上位9頭を◎○▲で分類
  └─ 予測結果を出力
```

## 2. 12軸スコアリング

各馬について、以下12軸を **0～100点** で評価します。最終スコアは12軸の平均値です。

| # | 軸記号 | 名称（日本語） | 説明 |
|---|--------|---|---|
| 1 | CF | キャリア形成 | 血統データの全成績（1着率 + 着数）から算出 |
| 2 | BF | 血統適性 | 種牡馬と母の父の統計的成績から複合スコア |
| 3 | SI | スピードインデックス | 上がり3Fタイム（速いほど高点） |
| 4 | CL | クラス適応性 | クラスコード別の勝率 + 補正 |
| 5 | JT | ジョッキー | 騎手別の成績統計 |
| 6 | PD | ペースデザイン | 通過順（1～4F）から位置取り安定性を評価 |
| 7 | FR | フィジカルレディネス | 馬体重の安定性（現在は基本値50） |
| 8 | TR | トラック適応性 | 場所×馬場（芝・ダ）別の勝率 |
| 9 | BL | ベース力 | 人気順に基づく市場評価 |
| 10 | HP | ハンディキャップ適応 | 斤量と年齢のバランス |
| 11 | SPD | スピード能力 | 走破時計の距離別相対順位（z-score） |
| 12 | MK | マーケット | 人気順 + オッズから市場評価スコア化 |

### 軸別計算ロジック（抜粋）

**CF（キャリア形成）**
```
total_races = 全成績1着数 + 2着 + 3着 + 着外
total_wins = 全成績1着数
CF = min(100, (total_wins / total_races) × 100 + min(total_races / 50, 30))
```

**BF（血統適性）**
```
sire_score = win_rate × 100 + place_rate × 20
dam_sire_score = win_rate × 100 + place_rate × 20
BF = (sire_score + dam_sire_score) / 2
```

**SI（スピード指数）**
```
agari_3f = 上がり3Fタイム (秒)
SI = max(10, 100 - (agari_3f - 30) × 2)
```

**MK（マーケット）**
```
IF 人気順 ∈ [1, 5]:
  MK = 100 - 人気順 × 10
ELIF 人気順 ∈ [6, 10]:
  MK = 50 - (人気順 - 5) × 5
ELSE:
  MK = max(10, 30 - (人気順 - 10))
```

## 3. 4チーム合議制

各馬について、**4つの独立したチーム** が12軸を評価し、スコアを集約します。

| チーム | 記号 | 特性 | 目的 |
|--------|------|------|------|
| Upset Detection | I | アンダードッグ重視 | 人気順が低い有力馬を発掘 |
| Mainstream | O | 主流評価 | 市場評価（人気順）が高い信頼度を検証 |
| Utility | U | バランス重視 | 中立的で総合的な評価 |
| Special/Anomaly | S | 異常値検出 | データ外れ値や特殊要因を検出 |

### 合議メカニズム

1. **チーム別スコア計算**
   - 各チーム(I/O/U/S)が独立して12軸を計算
   - チームI-typeは `人気順 > 5` の場合にMKボーナス加算
   - チームO-typeは `人気順 ≤ 3` の場合にMKボーナス加算

2. **スコア平均化**
   ```
   最終スコア = (I-type + O-type + U-type + S-type) / 4
   ```

3. **レース単位の集約**
   - 出馬表の全馬について上記を実施
   - 最終スコアでソート（降順）
   - 上位1頭(◎)、2-4位(○)、5-9位(▲)を出力

## 4. 統計マスター

### 統計対象と件数（2015-2026全データ）

| 統計種 | 件数 | 最小レース数基準 |
|--------|------|---|
| 種牡馬別 | 652 | 10レース以上 |
| 母の父別 | 1,117 | 10レース以上 |
| 騎手別 | 251 | 50レース以上 |
| 調教師別 | 262 | 50レース以上 |
| クラス別 | 11 | 制限なし |
| トラック別 | 20 | 50レース以上 |
| 距離別 | 48 | 制限なし |

### 統計項目

各統計について以下を計算：
- `races`: 対象レース数
- `wins`: 勝利数
- `places`: 複勝数（着順≤3）
- `win_rate`: 勝率 = wins / races
- `place_rate`: 複勝率 = places / races
- 距離統計のみ: `avg_time`, `std_time`（走破時計の平均・標準偏差）

## 5. 実装ファイル

### `course_master_v70_engine.py`

**クラス: `CourseMASTERv70`**

#### 主要メソッド

| メソッド | 入力 | 出力 | 説明 |
|---------|------|------|------|
| `load_data(result_files, blood_file)` | 結果CSV群, 血統CSV | - | 全データを読み込み、クリーニング、結合 |
| `build_statistics()` | - | - | 7種類の統計マスターを構築 |
| `score_horse(horse_row, team_type)` | 馬データ, チーム種別 | float(0-100) | 単一馬の12軸スコアを計算 |
| `score_race(race_data)` | 出馬表 | dict | レースの全馬スコア（4チーム平均） |
| `predict_race(race_data, top_n=9)` | 出馬表 | dict | ◎○▲の着順予測結果 |
| `save(filepath)` | ファイルパス | - | エンジン状態をPickle保存 |
| `load(filepath)` | ファイルパス | - | 保存済みエンジンを読み込み |

#### 使用例

```python
from course_master_v70_engine import CourseMASTERv70

# エンジン初期化
engine = CourseMASTERv70()

# データ読み込みと統計構築
result_files = [
    '/mnt/project/2015_2016結果.csv',
    '/mnt/project/2017_2018結果.csv',
    # ...
    '/mnt/project/2024_2026結果.csv'
]
engine.load_data(result_files, '/mnt/project/20260217血統.csv')
engine.build_statistics()

# レース予測
race_df = df_all[df_all['レースID'] == '526121201']  # 特定レースを抽出
predictions = engine.predict_race(race_df)

# 出力
for mark, horses in predictions.items():
    for horse_name, score, idx in horses:
        print(f"{mark}: {horse_name:15} ({score:.1f})")

# エンジン保存
engine.save('/mnt/user-data/outputs/engine.pkl')
```

## 7. 検証方法

### 想定される検証パターン

1. **時系列検証（推奨）**
   - 各レース日ごとに「その時点までのデータのみ」で統計構築
   - その日のレースを予測
   - 実績と比較

2. **クラス別検証**
   - G1/G2/G3などレースグレード別の精度測定
   - クラスコード別の適用精度

3. **レース場別検証**
   - 東京、中山、京都などの場所別精度測定

4. **多角的評価指標**
   - 正解率（◎が1着か否か）
   - ◎+○の複勝率
   - ◎+○+▲の的中率
   - ケンドールのτ（順位相関）

## 9. 制限事項

### 既知の制約

1. **父馬名マッチ率 75.5%**
   - 古いレース（2015年以前）やデータ欠損馬がある
   - マッチしない馬はBF軸がデフォルト値(50)

2. **フィジカルレディネス(FR)軸**
   - 前走馬体重データが結果CSVに含まれないため、仮の値(50)を使用
   - 今後のデータ拡張で改善可能

3. **ペースデザイン(PD)軸**
   - 通過順が利用可能な場合のみ計算
   - 欠損データが多い場合がある

4. **統計最小化**
   - 種牡馬・母の父：10レース以上基準
   - 騎手・調教師：50レース以上基準
   - 基準以下のデータはデフォルト値(50)を使用

## 11. 参考情報

### v7.0の設計原則

1. **シンプル性**：過度な複雑性を避け、解釈可能な12軸構成
2. **堅牢性**：欠損データへのデフォルト値設定で安定性確保
3. **スケーラビリティ**：新データの追加が容易な設計
4. **再現性**：統計マスターの保存・復元で同一の予測を再現可能
