  html = result["html"]  # source: "playwright" | "webfetch" | "cache"

## v7.0 → v7.3 の変更点

### Fix1: 勝率変換式の修正（スコア計算の正確化）

- **CF軸**: `place_rate`補正を廃止 → `win_rate`のみ＋信頼度係数（10走未満は50点に引き戻す）
- **BF軸**: `place_rate*20`加算を廃止 → `win_rate*100`のみ（`min(100, ...)`でキャップ）
- **CL軸**: `+30`固定加算を廃止 → `win_rate*100`のみ
- **JT軸**: `+30`固定加算を廃止 → `win_rate*100`のみ
- **predict_race**: `score_to_win_prob()`メソッドを追加。softmax（温度T=5.0）で勝率確率に変換し `P(win) <= P(place)/3` の原則を適用

### Fix2: キャリア走数ペナルティ（v7.3新規）

- 1〜3走目: CF係数 **0.70**（実データ比で32%低）
- 4〜5走目: CF係数 **0.85**（実データ比で15%低）
- 6走以上: ペナルティなし（1.00）
- 未出走・未知馬: cf_score = **20**（旧: 30）

### Fix3: MK軸オッズ帯補正（rev2）

バックテスト（2024-2026 Grade 277R）の結果に基づき「過大評価を減点するのみ」に絞った。

| オッズ帯 | MK補正 | 理由 |
|---------|--------|------|
| ～2倍 | ×0.85 | 実勝率 < 市場期待 −0.038 |
| 2〜3倍 | ×0.90 | 実勝率 < 市場期待 −0.043 |
| 3倍超 | ×1.00（補正なし） | — |

> 3〜8倍の加点（×1.05）は「2人気→1人気への誤誘導」を引き起こしたため撤廃。
> 薄人気フラグ（1〜2人気×4倍超×0.80）も的中率悪化（−13.6%）により削除。

## 7軸スコアリング（Phase 3 Step 5以降の現行）

2026-04-03 Phase 3 Step 5にてBF/HPを除外（バックテスト重賞1,544Rで予測精度に寄与しないと確認）。

| 軸 | 名称 | 計算ベース | 重み | 備考 |
|----|------|-----------|------|------|
| CF | キャリア形成 | 血統CSV全成績（勝率＋走数） | ×2.0 | 走数ペナルティあり |
| SI | スピードインデックス | 上がり3Fタイム | ×2.0 | |
| JT | ジョッキー | 騎手別統計勝率 | ×2.0 | |
| SPD | スピード能力 | 走破時計のz-score | ×2.0 | |
| PD | ペースデザイン | 通過順（1〜4F）の位置取り | ×1.0 | |
| BL | ベース力 | 人気順ベース評価 | ×0.3 | |
| MK | マーケット | 人気順＋オッズ帯補正 | ×0.3 | |

最終スコア = 7軸重み付き平均 × 4チーム（I/O/U/S）の平均

**除外した5軸（統計・コードは引き続き保持）**:
- FR: 常に50点固定（前走馬体重データ未対応）
- CL: 同一レース内全馬が同じクラスコード → 全馬同値
- TR: 同一レース内全馬が同じコース条件 → 全馬同値
- BF: 重賞バックテストで着順との相関係数+0.020（逆効果）→ 2026-04-03除外
- HP: 重賞バックテストで着順との相関係数-0.030（ほぼ無効）→ 2026-04-03除外

**7軸化の効果（重賞1,544R）**:
- 9軸: 32.2% vs 7軸: **35.3%**（+3.1%改善）
- G2: 31.9% → 34.9%（+3.0%）/ G3: 29.9% → 34.2%（+4.3%）

## 出力フォーマット・購入推奨ルール（2026-05-25 策定）

### 予測出力形式（◎○▲形式）

予測結果は以下の形式で出力する（`grade_race_predictor.py` 実装済み）：

```
◎ 馬名（人気・オッズ・エッジ値）
○ 馬名（人気・オッズ・エッジ値）
○ 馬名（人気・オッズ・エッジ値）
○ 馬名（人気・オッズ・エッジ値）
▲ 馬名（人気・オッズ・エッジ値）
▲ 馬名（人気・オッズ・エッジ値）
…（最大9頭）

荒れ指数：XX.X（波乱の可能性あり／堅い展開）
購入判定：◎エッジ+0.XXX → ★推奨 or 見送り
```

- 判定ラベル: `★★強推奨`（穴馬戦略＋エッジ両方） / `★推奨`（エッジのみ） / `△様子見` / `✗見送り`
- 購入の要否・馬券種・投資額の決定は**ユーザーが行う**

### 単勝100円シミュレーション廃止（2026-05-25）

- システムが提供するのは「どの馬にオッズの歪みがあるか」の情報のみ
- 馬連◎○候補表示でも金額は提示せず「対象馬名のみ」を表示

### 収支追跡の変更

- `candidates_YYYYMMDD.json` での保存形式は変更なし（条件戦の単勝購入は引き続き記録）

## 週次運用フロー

### 【土曜・日曜 9:00以降】天気予報取得（馬場悪化リスク確認）

Playwright MCPで Yahoo天気（競馬場別ページ）にアクセスして、
当日開催全会場の午後（12〜17時）の天気予報を取得して。

**取得手順**:
1. 以下のURLにアクセスする（開催会場のみ確認すればOK）
   ```
   https://weather.yahoo.co.jp/weather/race/
   ```
   競馬場一覧ページが表示される。当日開催の会場名をクリック。
   ※ 会場ごとに時間別の天気・降水確率・降水量が表示される。

2. 各開催会場について、**12時・15時・18時**（または最も近い時間帯）の以下を読み取る：
   - 降水確率（%）
   - 降水量（mm）

3. 取得したデータを以下のコマンドで渡す：
   ```
   python weather_checker.py --weather '{"date":"YYYY-MM-DD","venues":[{"name":"中京","hourly":[{"hour":9,"prob":10,"mm":0.0},{"hour":12,"prob":30,"mm":0.0},{"hour":15,"prob":50,"mm":1.0},{"hour":18,"prob":40,"mm":0.5}]},{"name":"中山","hourly":[...]}]}'
   ```

**警告ルール**（スクリプトが自動判定して表示）:
- 午後最大降水確率 **50%以上** → 「⚠️ 馬場悪化リスクあり」と表示
- 午後合計降水量 **2mm以上** → 「⚠️ 馬場状態変化に注意」と表示
- **判断は人間が行う**（自動的な購入判断変更はしない）

**馬場変化リスク早見**:
| 条件 | 雨の影響 |
|------|---------|
| A（中京ダ1401〜1800m） | 馬場問わず購入可。雨でも見送りなし |
| B（福島ダ1700m）★ | 稍重・不良は見送り。**降水確率50%以上は特に注意** |
| C（新潟芝2000m） | 重馬場は見送り。雨量に注意 |
| D（中京芝〜1400m） | 不良のみ見送り。稍重・重は積極買い |

## 騎手×場所×距離帯コース相性補正 検証（2026-06-03）

**採否判定: ❌ 見送り（的中率-3.8%・G1は-16.0%悪化）**

### 検証概要

| 項目 | 内容 |
|------|------|
| 対象 | 2024年 Grade (G1/G2/G3) 131R |
| 補正元 | `jockey_specialty.lift`（0.5〜2.0クリップ済み） |
| 採否基準 | 的中率改善幅+1%以上 |

### 結果

| 指標 | 補正なし（現行） | 補正あり | 差 |
|------|-------------|---------|-----|
| ◎1着的中率 | **27.5%** | 23.7% | **-3.8%** |
| 単勝ROI | **127.2%** | 112.3% | **-14.9%** |
| G1的中率 | 28.0% | 12.0% | **-16.0%** |

### 原因

- `jockey_specialty` は「得意な組み合わせのみ」を格納しており、lift の最小値が1.5（全件1.5〜5.5）
- Grade重賞に適用すると、馬の質より騎手得意場所が過剰ブーストされ正しい◎が押し退けられる
- 特にG1では1人気馬が正しく◎になっているケースで、騎手補正により誤った馬に◎が移動

### 今後改善するなら

- `jockey_specialty` を双方向（lift < 1.0 も含む）のテーブルに再集計する
- 現行テーブルはフィルタリング済みで補正には不適

**スクリプト**: `build_jockey_course_stats.py` / `backtest_jockey_course_2024.py`
**出力**: `output/jockey_course_stats.json` / `output/backtest_jockey_course_2024.csv`
