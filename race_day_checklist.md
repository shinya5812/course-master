### STEP 1｜前日夜（金曜夜〜土曜朝）— 開催場所の確認

**確認先**: JRA公式サイト（[jra.go.jp](https://www.jra.go.jp)）→「今週の重賞」または「出馬表」

**チェック内容**:

```
  中京が開催 → 条件A（ダート）・条件D（芝）の可能性あり
  福島が開催 → 条件B（ダート1700m）の可能性あり
  新潟が開催 → 条件C（芝2000m）の可能性あり
```

- [ ] 該当会場があれば翌日以降の天気予報を確認しておく

---

### STEP 2｜当日朝（8:30〜9:00）— 馬場状態の確認

**確認先**: JRA公式サイト → 「馬場情報」または各場の「発走前馬場状態」

**確認タイミング**:

| タイミング | 内容 |
|-----------|------|
| **8:30〜9:00** | 当日朝の馬場状態（第1レース前）を確認。これが基準になる |
| **第1・2レース後** | 状態変化を再確認。「良→稍重」「稍重→重」の変化に注意 |
| **雨が降っている場合** | 各レース前に都度確認する（条件B・Cは馬場変化で判断が変わる） |

**馬場状態の変化パターン（注意点）**:
- 朝「稍重」でも午後に乾いて「良」になるケースあり（条件B復活）
- 朝「良」でも午後に雨で「重」になるケースあり（条件C見送りに変化）
- **判断は対象レースの発走時刻直前の状態を優先する**

### STEP 4｜血統カテゴリ確認

**course_master.dbで確認する方法**（スクリプト）:

```python
# blood_category_check.py
# -*- coding: utf-8 -*-
import sqlite3, os
BASE_DIR = os.path.dirname(os.path.abspath(__file__))
db_path = os.path.join(BASE_DIR, 'course_master.db')

# ★ 対象馬リスト: (レース番号, 馬番, 馬名, 父馬名) を記入
CANDIDATES = [
    (6,  1, 'ヴェンデボヌール',       'ダノンスマッシュ'),
    (6,  5, 'トウカイジーク',         'マクフィ'),
    (6,  6, 'ケンブリッジグレイ',     'マインドユアビスケッツ'),
    # 必要な馬を追加...
]

TARGET_CATEGORY = '速力系'  # 条件Dの場合。条件A→マイラー系, B/C→スタミナ系

conn = sqlite3.connect(db_path)
cur  = conn.cursor()

print(f'\n血統カテゴリ照合結果（対象: {TARGET_CATEGORY}）')
print('-' * 60)
for race_no, horse_no, horse_name, sire_name in CANDIDATES:
    cur.execute("SELECT category FROM blood_category WHERE sire_name = ?", (sire_name,))
    row = cur.fetchone()
    if row:
        category = row[0]
        hit = '✅ 該当' if category == TARGET_CATEGORY else '─'
        print(f'{race_no}R 馬番{horse_no:2d} {horse_name:<20} 父:{sire_name} → {category} {hit}')
    else:
        print(f'{race_no}R 馬番{horse_no:2d} {horse_name:<20} 父:{sire_name} → 未登録（対象外）')

conn.close()
```

**カテゴリ対応**:

| カテゴリ | 対応条件 |
|---------|---------|
| マイラー系 | 条件A |
| スタミナ系 | 条件B・C |
| 速力系 | 条件D |
| 未分類 | **対象外**（購入しない） |

### STEP 5｜購入判断フロー

```
【条件Bチェック（最優先）】

  福島ダ1700m が開催？
  ├─ NO → 条件B スキップ
  └─ YES
       └─ 馬場は「良」または「重」？
            ├─ NO（稍重・不良） → 条件B 全レース見送り
            └─ YES
                 └─ 10番人気以上の馬の父馬がスタミナ系？
                      ├─ NO → その馬はスキップ
                      └─ YES → ✅ 条件B 購入対象（重み3）


【条件A・Dチェック（中京開催時）】

  中京が開催？
  ├─ NO → 条件A・D スキップ
  └─ YES
       ├─ 【条件A】ダート1401〜1800m の7〜9番人気馬で父馬がマイラー系？
       │    └─ YES（馬場問わず） → ✅ 条件A 購入対象（重み1）
       │
       └─ 【条件D】芝1200m/1400m の7〜9番人気馬で父馬が速力系？
            └─ 馬場が「不良」？
                 ├─ YES → 条件D 全レース見送り
                 └─ NO（良・稍・重） → ✅ 条件D 購入対象（重み1）


【条件Cチェック（新潟開催時）】

  新潟が開催？
  ├─ NO → 条件C スキップ
  └─ YES
       └─ 今月は4月または7月？
            ├─ YES → 条件C 全レース見送り（月フィルター）
            └─ NO
                 └─ 芝2000m の10番人気以上馬で父馬がスタミナ系？
                      └─ 馬場が「重」？
                           ├─ YES → 見送り
                           └─ NO（良・稍・不） → ✅ 条件C 購入対象（重み1）
```

## 4. 投資配分ルール

### 重み付け（単位ベット）

| 条件 | 重み | 1単位を100円とした場合 |
|------|------|----------------------|
| A（中京ダ） | 1 | 100円 |
| **B（福島ダ）** ★ | **3** | **300円** |
| C（新潟芝） | 1 | 100円 |
| D（中京芝） | 1 | 100円 |

### 投資上限の考え方

- **同一レースで複数馬が条件に該当した場合 → 全頭を各100円単勝で購入する**（分散ではなく全頭フルベット）
- 同一レース複数該当でも予算上限は「該当頭数 × 100円」（例: 2頭 → 200円）
- 1日の最大ベット数は対象レース数×馬数に依存するが、**条件B以外は多くて数頭程度**
- 「該当なし」の週は見送り。無理に他条件で代替しない

### 購入候補の出力フォーマット（必須項目）

購入候補を記録・出力する際は **必ず馬番を含める**。最低限の項目：

```
R番号 | 馬番 | 馬名 | 人気 | 単勝オッズ | 父馬 | 条件種別
例: 9R | 馬番8 | メイケイバートン | 9番人気 | 18.8倍 | ミッキーアイル | 条件D
```

### 券種

- **単勝**を基本とする（バックテストは全て単勝ベースで算出）
- 複勝は精度確認データなし（オッズデータ未入手のため）
