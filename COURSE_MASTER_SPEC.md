## 1. システム概要

JRA競馬の着順予測システム。過去レース結果（2015〜2026年・約54万件）と血統データから統計を構築し、7軸のスコアリングと4チーム合議制によって本命（◎）・対抗（○3頭）・単穴（▲5頭）を予測する。

### 主要ファイル一覧

| ファイル | 役割 |
|---------|------|
| `course_master_v73_engine.py` | 予測エンジン本体（現行） |
| `course_master_v70_engine.pkl` | 学習済みモデル（v7.3が後方互換で読み込む） |
| `prediction_app.py` | Flask製Webアプリ（`http://localhost:5000`） |
| `predict_future.py` | 未来レース予測（前走データ自動補完） |
| `grade_race_predictor.py` | 重賞専用予測＋荒れ指数算出 |
| `result_checker.py` | 購入候補の保存・結果照合・DB記録 |
| `backtest_grade_races.py` | 重賞バックテストスクリプト |
| `rebuild_pkl.py` | pkl再生成スクリプト |
| `update_race_results.py` | race_resultsテーブル追加インポート |
| `race_day_checklist.md` | 当日馬場確認・条件A〜D照合チェックシート |
| `course_master.db` | SQLiteデータベース（約23MB） |
| `20260217血統.csv` | 血統データ（51,111馬・cp932） |

### データ更新手順

1. TARGET JVからCSVを出力（cp932形式）し、プロジェクトフォルダに配置
2. `rebuild_pkl.py` を実行して統計再構築・pkl再生成
3. `prediction_app.py` を再起動

```bash
python rebuild_pkl.py
# → course_master_v70_engine.pkl を上書き更新
```

## 3. 予測エンジン仕様（v7.3・7軸）

### クラス構成

```
CourseMASTERv73
├── load_data()         # CSV・血統ファイル読み込み
├── _clean_data()       # データクリーニング・血統結合
├── build_statistics()  # 統計マスター構築（10セクション）
├── score_horse()       # 1頭の7軸スコア計算
├── score_race()        # 1レース全馬スコア計算（4チーム合議）
├── score_to_win_prob() # スコア → 推定勝率変換（softmax）
├── predict_race()      # レース予測（◎/○/▲を返す）
├── save()              # pkl保存
└── load()              # pkl読み込み
```

### 7軸スコアリング

#### 軸一覧・計算ロジック・重み

| 軸 | 名称 | 重み | 計算ロジック | 欠損時デフォルト |
|----|------|------|------------|----------------|
| **CF** | キャリア形成 | ×2.0 | 血統CSV全成績から勝率算出。信頼度係数（走数÷10、上限1.0）で10走未満は50点に引き戻す。走数ペナルティあり（下記参照）。 | 20点（血統CSV未掲載馬） |
| **SI** | スピードインデックス | ×2.0 | `max(10, 100 - (上がり3Fタイム - 30) × 2)` | 50点 |
| **JT** | ジョッキー | ×2.0 | `min(100, 騎手通算勝率 × 100)`。50走以上の騎手のみ統計化。 | 50点 |
| **SPD** | スピード能力 | ×2.0 | 同距離の走破時計をz-score化。`50 - z × 10`（クリップ: 10〜100）。 | 50点 |
| **PD** | ペースデザイン | ×1.0 | 前走通過順（1〜4F）の平均。`max(10, 100 - |平均通過順 - 7| × 3)` | 50点 |
| **BL** | ベース力 | ×0.3 | `max(10, 100 - 人気順 × 5)` | 50点 |
| **MK** | マーケット | ×0.3 | 人気順ベーススコア＋チーム特性調整＋オッズ帯補正（後述） | 50点 |

**除外軸（コード・統計は保持）**:

| 軸 | 除外理由 | 除外時期 |
|----|---------|---------|
| FR | 前走馬体重データ未対応・常に50点固定 | Phase 3 Step 1 |
| CL | 同一レース内で全馬同値 | Phase 3 Step 1 |
| TR | 同一レース内で全馬同値 | Phase 3 Step 1 |
| BF | 重賞1,544R着順相関係数 +0.020（逆効果） | Phase 3 Step 5 |
| HP | 重賞1,544R着順相関係数 -0.030（ほぼ無効） | Phase 3 Step 5 |

#### CF軸：走数ペナルティ係数

| 走数 | 係数 | 適用方法 |
|------|------|---------|
| 1〜3走 | 0.70 | `cf = 50 + (cf - 50) × 0.70`（50点超の部分のみ圧縮） |
| 4〜5走 | 0.85 | 同上 |
| 6走以上 | 1.00 | ペナルティなし |
| 未出走 | — | cf = 20点（固定） |

#### MK軸：計算ステップ

```
1. 人気順ベーススコア
   1〜5番人気:  mk = 100 - 人気 × 10
   6〜10番人気: mk = 50 - (人気-5) × 5
   11番人気以下: mk = max(10, 30 - (人気-10))

2. チーム特性調整
   チームI（Upset）: 6番人気以下に +(人気-5)×2 加算
   チームO（Mainstream）: 1〜3番人気に +(4-人気)×5 加算

3. オッズ帯補正（過大評価の減点のみ）
   〜2倍:   mk = 50 + (mk - 50) × 0.85
   2〜3倍:  mk = 50 + (mk - 50) × 0.90
   3倍超:   補正なし
```

### スコアリングフロー

```
入力: 出馬表データ（DataFrame）
         ↓
【Step 1】score_horse() × 全頭 × 4チーム（I/O/U/S）
  各チームで7軸を算出 → 重み付き平均でチームスコアを計算
         ↓
【Step 2】score_race()
  4チームスコアの単純平均 → 馬ごとの最終スコア
         ↓
【Step 3】score_to_win_prob()（softmax変換）
  softmax（温度T=5.0） → 推定勝率
  P(win) ≤ P(place)/3 の制約を適用（血統CSVの複勝率から上限算出）
  正規化（合計=1.0）
         ↓
出力: スコア順に ◎1頭 / ○3頭 / ▲5頭
```

### 4チームの役割

| チーム | 名称 | 特性 |
|--------|------|------|
| I | Upset（波乱狙い） | MK軸で人気薄を加点 |
| O | Mainstream（本命重視） | MK軸で上位人気をさらに加点 |
| U | Utility（汎用） | 調整なし |
| S | Special（特殊） | 調整なし |

最終スコア = 4チームスコアの**単純平均**

```bash
python result_checker.py --save '{"venue":"会場名","candidates":[[R,馬番,"馬名","条件",オッズ],...]}'

# 購入候補がゼロの場合も必ず保存する（結果照合スクリプトの起動に必要）
python result_checker.py --save '{"venue":"中京","candidates":[]}'
```

保存先: `candidates_YYYYMMDD.json`

#### STEP 4：重賞予測（grade_race_predictor.py）

Playwright MCPでJRA公式から重賞出馬表・オッズを取得して実行する。

```bash
# Windows環境では exitcode 49 回避のため一時ファイル経由で実行
# _run_XXXX_pred.py に sys.argv をセットして exec() する
```

出力内容:
- 荒れ指数（0〜100）とラベル
- ◎/○/▲の予測馬・スコア・推定勝率
- 全馬スコア上位10頭

#### STEP 5：レース後の結果照合

```bash
python result_checker.py --results '{"8":[馬番,"馬名",オッズ],...}'
# candidates_YYYYMMDD.json を自動読み込み
# condition_bet_history テーブルに記録
```

## 6. 荒れ指数仕様

`grade_race_predictor.py` の `calc_chaos_index()` で算出。0〜100のスコアでレースの荒れやすさを数値化する。

### 計算式（3要素の加重平均）

| 要素 | 重み | 計算方法 |
|------|------|---------|
| A（オッズ分散） | 40% | `min(100, (1番人気オッズ ÷ 3番人気オッズ) × 100)` |
| B（人気集中度） | 40% | `max(0, min(100, (0.50 - 1番人気支持率) ÷ 0.20 × 100))` |
| C（馬場状態） | 20% | 良=20 / 稍重=50 / 重=80 / 不良=100 |

### ラベル区分

| スコア範囲 | ラベル |
|-----------|--------|
| 0〜30 | 安定 |
| 30〜50 | やや波乱含み |
| 50〜70 | 波乱の可能性あり |
| 70〜85 | 大波乱注意 |
| 85〜 | 大波乱注意（見送り推奨） |

### 検証実績（2026-03-29）

| 重賞 | 荒れ指数 | 実際の結果 | 判定 |
|------|---------|-----------|------|
| 高松宮記念GⅠ | 79.0「大波乱注意」 | 2着に15番人気67倍が突入 | ✅ 的中 |
| マーチステークスGⅢ | 82.8「大波乱注意」 | 1〜3着が全員5番人気以下 | ✅ 的中 |
