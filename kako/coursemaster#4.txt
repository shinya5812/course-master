# COURSE MASTER — 次フェーズの作業方針
## 目的
予測精度の向上と、作業効率の構造的改善を同時に進める。
具体的には「毎週の予測サイクルを自動化・定型化」「並列分析で検証速度を上げる」「エンコーディング起因のエラーを仕組みで潰す」の3点を今月中に確立する。
---
## セッション開始時の確認事項
CLAUDE.md と PROGRESS.md を必ず読み込み、前回の状態を把握してから作業を開始すること。
---
## アクション①：/weekly スキルの作成（最初のセッションで完了）
毎週の競馬予測作業を定型化するため、.claude/skills/weekly/SKILL.md を新規作成すること。
内容は以下の手順を記載：
1. JRAの出馬表をPlaywright MCPで取得（403の場合はJSクリックナビゲーションで迂回）
2. grade_race_predictor.py を実行して予測を生成
3. 予測結果をPROGRESS.mdに記録
4. レース結果取得後はresult_checker.pyで的中照合
5. 的中率・回収率をPROGRESS.mdに追記
6. セッション終了前に git add -A && git commit -m "weekly: YYYY-MM-DD予測結果"
作成後、実際に /weekly を呼び出して動作確認すること。
---
## アクション②：Agent並列実行による分析高速化
単一セッションで逐次処理していた分析を並列化し、検証サイクルを短縮する。
以下のプロンプトを週次の予測セッションで使用すること：
「Taskツール（サブエージェント）を使って以下を並列実行してください。
Agent 1：東京・中山・阪神・京都のコース×距離別の複勝回収率をSQLite MCPで集計し、ROI上位10件を出力
Agent 2：blood_categoryごとの距離別勝率を集計し、距離カテゴリ（短距離/マイル/中距離/長距離）×血統系統の収益マトリクスを作成
Agent 3：直近2026年のレース結果から、ウォークフォワード検証を実施し各条件の年別成績を出力
3つのAgentが完了後、結果を統合してPROGRESS.mdに追記し、git commit まで完了させること。」
---
## アクション③：PostToolUseフック設定によるエラー削減
CP932/UTF-8混在によるCommand Failedを構造的に防ぐため、
~/.claude/settings.json にPostToolUseフックを追加すること：
- Editツール使用後、編集ファイルが .py の場合 → PYTHONIOENCODING=utf-8 で構文チェック自動実行
- Writeツール使用後、ファイルが .bat の場合 → 非ASCII文字検出 → 警告表示
設定後、テスト用 .py と .bat を1つずつ作成してフック動作確認まで完了すること。
---

1. PROGRESS.md に「何をしたか・何が残っているか・ブロッカー」を更新
2. git add -A && git commit（コミットメッセージは日本語可）
3. CLAUDE.md の Progress log を更新

## Step 1: Multi-Clauding対応orchestratorの実装

scripts/orchestrator.py を以下の構成に書き換えて：

```python
import subprocess
import json
import sys
from datetime import datetime
from pathlib import Path

def run_claude_agent(prompt: str, output_file: str) -> subprocess.Popen:
    """独立したclaudeプロセスを起動して返す"""
    return subprocess.Popen(
        ["claude", "-p", prompt, "--output-format", "text"],
        stdout=open(output_file, "w", encoding="utf-8"),
        stderr=subprocess.PIPE
    )

def main(date: str):
    base = Path("C:/Users/shiny/Dropbox/shinya_wa/coursemaster")
    
    # 3つの独立したClaudeプロセスを同時起動
    processes = {
        "predictor": run_claude_agent(
            f"grade_race_predictor.pyを実行して{date}の予測をpredictions_{date}.jsonに保存して",
            str(base / f"predictions_{date}_log.txt")
        ),
        "analyst": run_claude_agent(
            f"DBから{date}の条件別ROIを集計してanalysis_{date}.jsonに保存して",
            str(base / f"analysis_{date}_log.txt")
        ),
        "result_checker": run_claude_agent(
            f"result_checker.pyを実行して直前レースのP&Lをpnl_{date}.jsonに保存して",
            str(base / f"pnl_{date}_log.txt")
        ),
    }
    
    print(f"[Multi-Clauding] 3プロセス起動完了: {list(processes.keys())}")
    
    # 全プロセス完了待機
    results = {}
    for name, proc in processes.items():
        proc.wait()
        results[name] = "success" if proc.returncode == 0 else "failed"
        print(f"  [{name}] {results[name]} (exit={proc.returncode})")
    
    # 統合レポート生成
    report_path = base / f"weekly_report_{date}.md"
    with open(report_path, "w", encoding="utf-8") as f:
        f.write(f"# Weekly Report {date}\n")
        f.write(f"## Multi-Clauding実行結果\n")
        for name, status in results.items():
            f.write(f"- {name}: {status}\n")
    
    print(f"[完了] {report_path}")

if __name__ == "__main__":
    date = sys.argv[1] if len(sys.argv) > 1 else datetime.today().strftime("%Y-%m-%d")
    main(date)
```
